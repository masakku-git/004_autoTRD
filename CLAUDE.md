# 004_autoTRD プロジェクトルール

## 戦略ファイルの管理

**戦略の変更は既存ファイルの修正ではなく、新規ファイルの作成で行う。**

- 既存の戦略ファイル（例: `breakout_v2.py`）は変更しない
- 改良・修正は新しいバージョンファイルとして作成する（例: `breakout_v3.py`）
- 旧バージョンはそのまま残し、比較・ロールバックができるようにする

### ファイル命名規則

```
src/strategy/plugins/<strategy_name>_v<N>.py
```

例:
- `sma_crossover_v1.py` → 変更せずに残す
- `sma_crossover_v2.py` → 変更せずに残す
- `sma_crossover_v3.py` → 新規作成（改良版）

### 理由

- バージョン間のシミュレーション比較が容易になる
- 変更履歴がファイル単位で追跡できる
- ロールバックが安全・確実に行える
